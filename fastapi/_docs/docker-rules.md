# Docker 컨테이너 생성 규칙

## 중복 생성 금지 — 생성 전 기존 리소스 확인 필수

DB · 백엔드 컨테이너를 새로 만들어 달라는 요청을 받아도, **곧바로 생성하지 않는다.**
먼저 동일하거나 같은 역할을 하는 리소스가 이미 떠 있는지 확인하고, 있으면 사용자에게
알리고 승인을 받은 뒤에만 만든다.

### 확인 순서

1. `docker compose ps` / `docker ps -a` 로 현재 떠 있는(또는 멈춘) 컨테이너 목록 확인
2. `docker-compose.yaml`에 같은 이름·같은 이미지·같은 역할의 서비스가 이미 정의돼 있는지 확인
3. 겹치는 게 있으면:
   - 기존 것을 그대로 쓸지
   - 기존 것을 내리고 새로 만들지
   - 이름을 바꿔서 별개로 둘지
   사용자에게 묻고 승인받은 뒤 진행한다.
4. 겹치는 게 없을 때만 바로 생성한다.

### Why

과거에 확인 없이 반복 생성하다 보니 `my-fastapi`, `my-www`, `my-web-server` 같은
정체불명의 중복 nginx 컨테이너, 그리고 서로 다른 자격증명으로 같은 도메인을 가리키는
중복 cloudflared 터널이 계속 쌓였다. 전부 뒤늦게 발견해서 정리해야 했다.

### 적용 대상

- DB 컨테이너 (pgvector, neo4j, redis 등)
- 백엔드 컨테이너 (backend, 그 외 앱 서버)
- 그 외 도메인/포트를 점유하는 서비스(cloudflared 터널 등)도 동일하게 적용한다 —
  같은 호스트네임을 라우팅하는 터널을 새로 만들기 전에 기존 터널이 있는지 먼저 확인한다.
